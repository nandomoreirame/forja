# PRD-001: New Session Type Selector Dialog

> **Status:** approved
> **Author:** Claude (Sonnet 4.5)
> **Created:** 2026-02-22
> **Last Updated:** 2026-02-22

---

## Overview

### Problem Statement

Currently, when users press Ctrl+T or click "New Session" in the command palette, a new Claude Code session is created immediately. There's no way to choose between creating a Claude Code session or a regular terminal session. This limits the flexibility of the application, as users may want to run standard shell commands without invoking Claude Code.

### Goal

Provide users with a choice between creating a "Claude Code" session or a "Regular Terminal" session through an intuitive modal dialog.

### Success Metrics

- [ ] 100% of "New Session" actions show the session type selector
- [ ] Users can successfully create both Claude Code and regular terminal sessions
- [ ] No regression in existing tab/session functionality
- [ ] Modal follows Catppuccin Mocha design system guidelines

---

## Background

### Context

Forja is a dedicated desktop GUI client for Claude Code built with Tauri 2 + React. The current implementation automatically creates Claude Code sessions, but users need the ability to create regular terminal sessions for standard shell operations.

### Current State

- Ctrl+T creates a new Claude Code session immediately
- "New Session" command in palette creates a Claude Code session immediately
- No UI for session type selection
- Terminal tabs system already exists and works well

### Research & Insights

- Users expect terminal applications to support both AI-assisted and regular shell sessions
- Modal dialogs are a familiar pattern for making choices before creating resources
- The existing keyboard shortcut (Ctrl+T) is intuitive and should be preserved

## User Experience

### Target Users

1. **Developers using Claude Code:** Users who want AI-assisted terminal sessions
2. **Power users:** Users who want both AI and regular shell access

### User Flow

```
User presses Ctrl+T OR clicks "New Session"
  → Modal appears with two options
    → User clicks "Claude Code"
      → New Claude Code tab created
      → Modal closes
    → User clicks "Terminal"
      → New regular terminal tab created
      → Modal closes
    → User presses Escape OR clicks outside
      → Modal closes
      → No tab created
```

### Wireframes/Mockups

Modal should have:

- Title: "New Session"
- Subtitle: "Choose session type"
- Two large, clickable cards:
  - "Claude Code" (with sparkles icon)
  - "Terminal" (with terminal icon)
- Dark theme (Catppuccin Mocha)
- Centered on screen

## Scope

### In Scope

- New session type selector modal component
- State management for modal visibility
- Integration with existing tab creation system
- Keyboard shortcuts handling (Ctrl+T, Escape)
- Visual design following design system
- Unit tests for modal component
- Integration tests for session creation flow

### Out of Scope

- Changes to terminal session rendering logic
- Changes to Claude Code integration
- Session persistence/restoration
- Session templates or presets
- Multi-session creation
- Session type switching after creation

### Assumptions

- Tauri backend already supports regular terminal sessions
- Tab system can handle both session types
- No breaking changes to existing API

### Constraints

- Must use existing shadcn/ui Dialog component
- Must integrate with existing Zustand stores
- Must maintain current keyboard shortcut behavior
- Must not break existing tests
