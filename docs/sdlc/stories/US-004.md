# US-004: Workspace Switcher in Topbar

**Status:** Ready for Dev
**PRD:** PRD-004
**Date:** 2026-03-15

---

## User Story

**As a developer using Forja with multiple projects,**
**I want a visual workspace switcher in the topbar**
**so that I can instantly see which workspace is active, switch between workspaces,
and manage workspace names/colors/icons without losing any of my running sessions.**

---

## Acceptance Criteria

### AC-1: Workspace button in topbar

```
Given: Forja is open with at least one workspace
When:  I look at the topbar left cluster
Then:  I see a small colored icon button representing the active workspace
And:   Hovering shows a tooltip with the workspace name
```

### AC-2: Dropdown opens on click

```
Given: I click the workspace button
When:  The dropdown opens
Then:  I see a "Switch workspace" title
And:   All workspaces are listed with their icon and name
And:   The active workspace row is visually highlighted
```

### AC-3: Inline editing of active workspace

```
Given: The dropdown is open
When:  I click the pencil (edit) button on the active workspace
Then:  A name input field appears (pre-filled with current name)
And:   A color picker shows 7 colored circles
And:   An icon picker shows 14 icons in a grid
```

### AC-4: Save workspace changes

```
Given: I am in edit mode for a workspace
When:  I change the name and click the checkmark (confirm) button
Then:  The workspace is updated (name, color, icon)
And:   The topbar button reflects the new color and icon immediately
And:   Changes persist after app restart
```

### AC-5: Delete workspace

```
Given: The dropdown is open
When:  I click "Delete workspace" (red link)
Then:  A confirmation dialog appears
And:   If I confirm, the workspace is deleted
And:   If I cancel, nothing changes
```

### AC-6: Create new workspace

```
Given: The dropdown is open
When:  I click "+ Create new workspace"
Then:  A new Forja window opens
And:   The current window remains fully operational
And:   All PTY sessions in the current window continue running
```

### AC-7: Switch workspace

```
Given: Multiple workspaces exist
When:  I click an inactive workspace row in the dropdown
Then:  That workspace becomes active
And:   Its projects are loaded in the sidebar
And:   The topbar icon updates to show the new workspace's color/icon
```

### AC-8: Default workspace values

```
Given: An existing workspace without color/icon fields
When:  It is rendered
Then:  It defaults to color="mauve" and icon="layers"
```
