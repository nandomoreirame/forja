# PRD-002: Browser Screenshot to Clipboard

**Status:** Approved
**Author:** Development Team
**Date:** 2026-03-10
**Version:** 1.0

---

## Problem Statement

Users of Forja's embedded browser pane often need to capture the current state of a webpage to share with AI assistants (Claude, Gemini, etc.) or save for documentation. Currently, there is no built-in way to capture a screenshot directly from the browser toolbar.

## Goals

1. Provide a one-click screenshot capture of the embedded browser webview
2. Save the screenshot directly to the system clipboard for immediate paste
3. Provide visual feedback (success/error) to the user
4. Follow existing UI patterns and design system (Catppuccin Mocha)

## Non-Goals

- Saving screenshots to disk (out of scope for MVP)
- Partial/region selection screenshots
- Screenshot history or management
- Annotating screenshots

## User Stories

- **US-002-1:** As a developer, I want to capture the current browser page as a screenshot so I can paste it into an AI conversation
- **US-002-2:** As a developer, I want visual feedback after taking a screenshot so I know if it was successful

## Requirements

### Functional Requirements

| ID | Requirement | Priority |
|----|-------------|----------|
| FR-1 | Camera icon button in browser toolbar, next to close button | Must |
| FR-2 | Clicking captures the webview content as PNG | Must |
| FR-3 | Screenshot is written to system clipboard | Must |
| FR-4 | Visual feedback: button turns green briefly on success | Must |
| FR-5 | Error handling: toast/console error if capture fails | Should |

### Technical Requirements

| ID | Requirement |
|----|-------------|
| TR-1 | Use Electron's `webContents.capturePage()` for webview capture |
| TR-2 | Use Electron's `clipboard.writeImage()` to write NativeImage to clipboard |
| TR-3 | IPC channel: `browser:screenshot` (invoke pattern) |
| TR-4 | Frontend calls IPC via `webviewRef.current.getWebContentsId()` |
| TR-5 | Preload must expose `browser.screenshot()` method |

## Technical Approach

### IPC Flow

```
Frontend (BrowserPane)
  → calls invoke("browser:screenshot", { webContentsId })
  → Electron main.ts handler
  → webContents.capturePage() → NativeImage
  → clipboard.writeImage(image)
  → returns { success: true }
Frontend
  → shows visual success feedback (green check, 2s)
```
