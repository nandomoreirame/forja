# PRD-003 — Context Menu e Renomear Tab de Terminal

**Status:** Draft
**Data:** 2026-03-10
**Author:** Claude Code

---

## 1. Visão Geral

Adicionar um menu de contexto (right-click) nas tabs de terminal da TabBar com as opções:
1. **Edit tab** — permite ao usuário renomear inline a tab diretamente no texto da tab
2. **Fechar tab** — fecha a tab atual (equivalente ao botão X já existente)

## 2. Problema

Atualmente, as tabs de terminal possuem apenas um botão X para fechar. Não há forma de:
- Renomear uma tab para identificá-la mais facilmente (ex: "Build", "Tests", "Backend")
- Acessar ações de gerenciamento de forma contextual

## 3. Objetivos

- Melhorar a identificação visual de múltiplas tabs abertas
- Oferecer uma UX familiar com menus de contexto
- Permitir nomes customizados que persistem durante a sessão

## 4. User Stories

- **US-003-1:** Como usuário, quero clicar com botão direito em uma tab para ver opções de gerenciamento
- **US-003-2:** Como usuário, quero renomear uma tab inline para identificá-la melhor
- **US-003-3:** Como usuário, quero fechar uma tab pelo menu de contexto

## 5. Requisitos Funcionais

### RF-001: Context Menu nas Tabs
- Right-click em qualquer tab abre um context menu
- Menu exibe as opções: "Edit tab" e "Fechar tab"
- "Fechar tab" tem estilo destrutivo (vermelho)

### RF-002: Renomear Tab Inline
- Ao clicar "Edit tab", o texto da tab se transforma em um input
- Input é pré-preenchido com o nome atual
- Confirmar: Enter ou blur fora do input
- Cancelar: Escape
- Nome vazio não é aceito (mantém nome atual)
- Nome whitespace-only não é aceito

### RF-003: Exibição do Nome Customizado
- Tabs com nome customizado exibem o nome customizado em vez do nome automático
- Tabs sem nome customizado continuam com o comportamento dinâmico atual (ex: "Claude Code #1")

### RF-004: Persistência de Sessão
- Nome customizado persiste enquanto a sessão está aberta
- Ao fechar e reabrir a app, tabs começam sem nome customizado (sem persistência em disco nesta versão)

## 6. Requisitos Não-Funcionais

- A13y: context menu deve ser acessível via teclado
- UX: animação de abertura/fechamento do menu fluida
- Design: seguir o Design System Catppuccin Mocha

## 7. Escopo Fora

- Persistência do nome customizado em disco (versão futura)
- Atalho de teclado para renomear tab (versão futura)

## 8. Critérios de Aceite

- [ ] Right-click em tab abre context menu com 2 opções
- [ ] "Fechar tab" fecha a tab corretamente
- [ ] "Edit tab" ativa modo de edição inline na tab
- [ ] Enter/blur confirma o nome (se não vazio e diferente)
- [ ] Escape cancela a edição
- [ ] Nome customizado é exibido no lugar do nome automático
- [ ] Todos os testes passam
