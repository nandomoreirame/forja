# ADR-002: Browser Screenshot Implementation Strategy

**Status:** Accepted
**Date:** 2026-03-10
**Deciders:** Development Team
**PRD:** PRD-002

---

## Context

We need to implement screenshot capture for the embedded browser webview in Forja. The webview is rendered using Electron's `<webview>` tag. We need to decide the best approach to capture the webview content and write it to the clipboard.

## Decision Drivers

1. Security: must not expose `nodeIntegration` in renderer
2. Correctness: must capture the actual webview content (not the BrowserWindow)
3. Simplicity: minimal IPC surface area
4. Testability: must be unit-testable in jsdom environment

## Options Considered

### Option A: `webContents.capturePage()` via IPC (chosen)

The renderer passes the webview's `webContentsId` to the main process. The main process resolves the `WebContents` and calls `capturePage()`.

**Pros:**
- Secure (stays in main process)
- Accurate (captures exact webview content)
- Synchronous from user perspective

**Cons:**
- Requires IPC round-trip
- Need to pass webContentsId from renderer

### Option B: Canvas drawImage in renderer

Use `<canvas>` and draw the webview element directly.

**Pros:** No IPC needed

**Cons:**
- Cross-origin restrictions block this
- Doesn't work with `<webview>` tag reliably
- Security issues

### Option C: Screenshot of entire BrowserWindow

Use Electron's `BrowserWindow.capturePage()`.

**Pros:** Simple

**Cons:**
- Captures the entire app window, not just the webview
- User doesn't want to capture the toolbar/UI

## Decision

**Option A** - Use `webContents.capturePage()` via IPC.

The renderer will call `webviewRef.current.getWebContentsId()` to get the webview's webContentsId, pass it to the main process via `browser:screenshot` IPC channel. Main process uses `webContents.fromId(id)?.capturePage()` and `clipboard.writeImage()`.

## Implementation Notes

```typescript
// Main process (electron/main.ts)
import { clipboard } from "electron";

ipcMain.handle("browser:screenshot", async (_event, args: { webContentsId: number }) => {
  const wc = webContents.fromId(args.webContentsId);
  if (!wc) throw new Error("WebContents not found");
  const image = await wc.capturePage();
  clipboard.writeImage(image);
  return { success: true };
});
```

```typescript
// Frontend (browser-pane.tsx)
const handleScreenshot = useCallback(async () => {
  const wv = webviewRef.current;
  if (!wv) return;
  const webContentsId = wv.getWebContentsId();
  await invoke("browser:screenshot", { webContentsId });
  // show success feedback
}, []);
```

## Consequences

- **Positive:** Clean IPC pattern, secure, testable
- **Negative:** Webview must be loaded for `getWebContentsId()` to return a valid ID
- **Neutral:** Consistent with existing IPC patterns in the codebase
