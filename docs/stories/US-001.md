# US-001: Choose Session Type When Creating New Session

> **Status:** ready
> **Priority:** high
> **Story Points:** 3
> **PRD:** [PRD-001](../prds/PRD-001.md)
> **Created:** 2026-02-22

---

## User Story

**As a** Forja user,
**I want** to choose between creating a Claude Code session or a regular terminal session,
**So that** I can use AI assistance when needed or run standard shell commands when I don't need AI.

---

## Acceptance Criteria

### Scenario 1: Creating Session via Keyboard Shortcut

**Given** I have a project open in Forja
**When** I press Ctrl+T (or Cmd+T on macOS)
**Then** a modal appears asking me to choose between "Claude Code" or "Terminal"
**And** I can select either option to create the corresponding session type

### Scenario 2: Creating Session via Command Palette

**Given** I open the command palette (Ctrl+Shift+P)
**When** I select the "New Session" command
**Then** a modal appears asking me to choose between "Claude Code" or "Terminal"
**And** I can select either option to create the corresponding session type

### Scenario 3: Dismissing Modal with Keyboard

**Given** the new session modal is open
**When** I press the Escape key
**Then** the modal closes without creating a session

### Scenario 4: Dismissing Modal by Clicking Outside

**Given** the new session modal is open
**When** I click outside the modal area
**Then** the modal closes without creating a session

### Scenario 5: Creating Claude Code Session

**Given** the new session modal is open
**When** I click the "Claude Code" option
**Then** a new Claude Code tab is created
**And** the modal closes
**And** the new tab becomes active

### Scenario 6: Creating Regular Terminal Session

**Given** the new session modal is open
**When** I click the "Terminal" option
**Then** a new regular terminal tab is created
**And** the modal closes
**And** the new tab becomes active

- `frontend/stores/app-dialogs.ts` - Add `newSessionOpen` state
- `frontend/stores/terminal-tabs.ts` - Modify `addTab` to accept `sessionType` parameter
- `frontend/components/new-session-dialog.tsx` - New component
- `frontend/components/command-palette.tsx` - Update "New Session" command handler
- `frontend/App.tsx` - Update Ctrl+T keyboard shortcut handler

### API Changes

**`useAppDialogsStore`:**

```typescript
// Add to interface
newSessionOpen: boolean;
setNewSessionOpen: (open: boolean) => void;
```

**`useTerminalTabsStore`:**

```typescript
// Modify addTab signature
addTab: (id: string, path: string, sessionType?: 'claude-code' | 'terminal') => void;
```
